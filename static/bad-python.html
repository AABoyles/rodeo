Bad news! Rodeo can't start

What's going on?

Unfortunately Rodeo wasn't able to find the python command it needs in order to run. Rodeo uses the same libraries the IPython Notebook does. This means, if you don't have your notebook setup properly, Rodeo probably isn't going to work.

How do I fix it?

The good news is that we've identified the problem and it's fairly easy to fix. The root of the problem is that you don't have Python setup correctly (or it's setup in a pretty non-standard way that Rodeo doesn't recognize). You've got a couple options.

The "I just want it to work" option

If you just want this to work as quickly as humanly possible, you can install a pre-packaged version of Python.
The easiest way to go about this is to install Anaconda. It's free, you can double-click to install, and it has everything you need to use Rodeo.
Once you're done, try re-running Rodeo.

The "I know what I'm doing" option

Rodeo needs to be able to import either the jupyter_client or the IPython module. If you don't have one of these installed, then that's prboably your problem.
If you do have one of them installed, then the issue is that Rodeo is using the "wrong python". Rodeo defaults to using whatever python is on your PATH (or on Windows, whatever is set in your Environment Variables).
Not sure if you have python installed? You should probabyl go back to the "I just want it to work" section.

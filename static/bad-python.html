Bad news! Rodeo can't start

What's going on?

Unfortunately Rodeo wasn't able to find the python command it needs in order to run. Rodeo uses the same libraries the IPython Notebook does. This means, if you don't have your notebook setup properly, Rodeo probably isn't going to work.

How do I fix it?

The good news is that we've identified the problem and it's fairly easy to fix. The root of the problem is that you don't have Python setup correctly (or it's setup in a pretty non-standard way that Rodeo doesn't recognize). You've got a couple options.

(1) The "I just want it to work" option

If you just want this to work as quickly as humanly possible, you can install a pre-packaged version of Python.

(2) The "I know what I'm doing" option

Rodeo needs to be able to import either jupyter_client or IPython. If you don't have one of these installed, then that's prboably your problem.

If you have your python environment in a different location (you can test this by running which python), you can map that environment to one of the python paths that Rodeo supports: 

$ which python
# /some/other/python
$ mkdir -p ~/bin
$ ln -s `which python` ~/bin/python
